## M5 — 콘텐츠 & 폴리시 (코드 콘텐츠 완료 — 폴리시는 Unity 에디터 작업)

- [x] 몬스터 다양화 — 고블린(빈사 도주)/해골(단단·강타)/슬라임(약함·넓은 배회) 3종, 깊이 비례 혼합, 진행 중 추가 스폰(12턴 간격·활성 층 4마리 상한·시야 밖 스폰)
- [x] 아이템 다양화 — 6종: 물약/폭탄/냉기 폭탄 + 기름 병(불+기름 요소 반응 첫 실체화, `OilRules`)/투척 단검/귀환 두루마리. 분배 3:3:1:1:1:1, 표시 정보는 `ItemCatalog` 단일 출처
- [x] 밸런스 튜닝(1차) — C# 시뮬레이터(Core 재사용, 전술 그리디 300판)로 3던전 체인 측정.
  던전 전환 회복 50%(모닥불) 도입, 사냥꾼 HP 8, 연금 ATK 3 → 완주율 기사 49%/사냥꾼 43%/연금 25%(도구 미사용 하한). 이후 플레이 테스트로 재조정
- [x] UI 폴리시(부분) — 메인 메뉴·게임오버·인벤토리 화면 완료. 조합 화면은 남음
- [x] 상호작용 버튼 — 인접 문/폭발통 감지 시 컨텍스트 버튼(OPEN/CLOSE/PUSH) 표시, 탭과 같은 행동 경로(`PerformContextInteraction`) 사용
- [x] 조합 시스템 Core — 재료 3종(약초/화약/서리 수정) 스폰·창고 보관 + 레시피 3종(약초×2→물약, 화약×2→폭탄, 폭탄+서리 수정→냉기 폭탄), `CraftingRules` 단일 출처. 조합 UI는 별도 작업
- [x] 요소 반응 확장 — 물 웅덩이(`TileData.wet`, 생성기 배치) + 냉기→웅덩이 연쇄 결빙(`WaterRules.ChainFreeze`), 불 폭발→증발, 젖은 바닥엔 기름 안 붙음
- [ ] 사운드 (마지막으로 연기)
- [x] 아트 에셋 교체 테스트 (파이프라인 검증) — 액터/아이템 슬롯 3종(플레이어/고블린/물약)에 실제 PNG 연결 확인. 플레이스홀더는 `Tools/ArtPipeline/generate_actor_sprites.py`, AI 소스 확보 시 교체

- [x] 게임오버/승리 오버레이(사인 한글화·도달 층·처치) + 씬 리로드 재시작 (`RunSummary`)

- [x] 미니맵 실데이터화(활성 층 안개/적/아이템/플레이어)

- [x] 던전 전환 모닥불 회복 50% — 시뮬 근거(회복 0%=완주 18% → 50%=49%)
- [x] 상점 & 메타 해금 — 메인 메뉴 상점(소모품 구매 → 창고 적립 → 새 판 반입, `ItemCatalog.ShopPrice`), 영웅 해금(기사 기본, 사냥꾼 80G/연금술사 120G). extraction 루프 완성: 파밍 → 생환 → 골드 → 상점/해금 → 재도전
- [x] 층 규모 확대 — 층 크기 9~20(기본 13), 적/아이템/추가 스폰 상한이 면적 비례로 스케일(`DungeonGenerator.AreaSpawnBonus`)

## UI 방침 (확정, 코드는 M4/M5)

- UI 이원화 확정: **화면공간 평면 = UI Toolkit, 월드 앵커/추종 = UGUI**. 상세·화면별 배치·Claude 디자인 워크플로는 `docs/UI_ARCHITECTURE.md` (SSOT).
- 화면 회전·층·현재 위치·전투 모드용 검증 HUD는 UI Toolkit으로 구현했다. 인벤·조합·메뉴·메타·최종 상호작용 팝업은 M4/M5에서 이식한다.

## 확정된 방향 (GDD v0.2)

- **한 판 목표**: 최심층 도달 (계속 아래로 → 최심층/보스). 프로토 5~8층, 한 판 20~30분.
- **진행/죽음**: 로그라이트 메타 프로그레션 (죽으면 판 종료, 일부 해금 이월).
- **테마**: 정통 판타지 던전 (고블린/해골/슬라임/마법).
- **영웅**: 프로토는 단일 영웅으로 단순화, 이후 직업제 검토.
